\paragraph{Positioning.}
This section is written so that (a) the manuscript compiles immediately and (b) the planned figures and tables can be populated in-place as experiments complete. The scientific objective is to evaluate CAPO as a \emph{pure aggregation change} within RLVR. Accordingly, we hold constant: (i) the base policy and optimizer, (ii) the sampler/decoder and verifier, (iii) PPO/GRPO-style clipping and advantage normalization, and (iv) the per-step token budget. The only degrees of freedom across methods are the per-trajectory aggregation weights and any statistics required to compute them.

\subsection{Evaluation Protocol}
\label{subsec:eval-protocol}

\paragraph{Models and task.}
All experiments in this manuscript use a single open-weight backbone model, \textbf{Qwen2.5--1.5B--Instruct}, and a single RLVR-style verifiable task, \textbf{CountDown}. This choice reflects an A10-feasible evaluation target while preserving the scientific scope of the paper: CAPO is evaluated as a \emph{pure aggregation change} within RLVR. We retain two \emph{context-length regimes} to probe length drift and long-response weighting effects:
\begin{itemize}
  \item \textbf{Short-context regime:} max response length 2048.
  \item \textbf{Long-context regime:} max response length 8192.
\end{itemize}

\paragraph{Compute and budgeting.}
All methods are run under matched \emph{tokens-per-step} and matched \emph{total training tokens}. We report the exact token accounting (prompt tokens, sampled tokens, and any truncated/masked tokens) alongside the main results. The minimal four-day suite on an 8\,$\times$\,A10 node targets:
\begin{enumerate}
  \item \textbf{Primary suite (short-context):} Qwen2.5--1.5B with max response length 2048 on CountDown, for all baselines and CAPO, with $S=2$ seeds.
  \item \textbf{Long-context stress test:} Qwen2.5--1.5B with max response length 8192 on CountDown, for a focused subset (\(\Delta L\) best $\alpha$ and CAPO best settings), with $S=1$--$2$ seeds.
\end{enumerate}

\paragraph{Baselines.}
We compare the following aggregation rules, using the unified linear form in \S\ref{sec:setup} and the notation in \S\ref{sec:method}:
\begin{itemize}
  \item \textbf{GRPO Norm:} per-sample normalization by $L_i$ (sample-level length normalization).
  \item \textbf{DAPO Norm:} batch-level length normalization (total-length scaling).
  \item \textbf{Dr.~GRPO Norm:} length-agnostic normalization with a fixed constant.
  \item \textbf{$\Delta L$ Normalization:} $x_i \propto L_i^{-\alpha}$ with the recommended $\alpha\in\{0.75,1\}$ \citep{he2025deltal}.
  \item \textbf{CAPO:} covariance-aware precision weighting with sublinear dampening, $x_i \propto L_i^{-\alpha}/\widehat{v}_i$ with $\alpha\in(0,1)$.
\end{itemize}

\paragraph{Seeds.}
For the short-context primary suite we use $S=2$ independent seeds per method. For the long-context stress test we use $S=1$--$2$ seeds, depending on stability and wall-clock constraints.

\subsection{Primary Metrics and Reporting}
\label{subsec:metrics}

\paragraph{Accuracy metrics.}
We report:
\begin{itemize}
  \item \textbf{Avg@8} and \textbf{Pass@8} on CountDown (following \citep{he2025deltal} where applicable).
  \item \textbf{Best-of-run}, \textbf{last-$k$ average}, and \textbf{area-under-curve} (AUC) with respect to training steps and with respect to consumed tokens.
\end{itemize}

\paragraph{Stability metrics.}
We report stability using diagnostics motivated by the variance and CV analyses in \citet{he2025deltal}:
\begin{itemize}
  \item \textbf{Monotonicity:} Pearson correlation between step index and held-out accuracy (measured at fixed evaluation cadence).
  \item \textbf{Negative-drift rate (NDR):} fraction of evaluation intervals with negative held-out improvement relative to the preceding evaluation.
  \item \textbf{Weight distribution diagnostics:} per-step coefficient of variation and kurtosis of the normalized weights $\{w_i\}$.
  \item \textbf{Length equity:} stratified performance and contribution by response-length deciles.
\end{itemize}

\paragraph{Efficiency metrics.}
We report tokens-to-target (TTT) for pre-specified thresholds on held-out metrics and wall-clock overhead for CAPO statistics.

\subsection{Minimum Experiment Matrix}
\label{subsec:min-matrix}

The following matrix is designed to be feasible within one week while still meeting ICML expectations for completeness and clarity.

\\paragraph{E1: Main comparison (short-context).}\nFor Qwen2.5--1.5B at max response length 2048, run all baselines and CAPO on CountDown under matched tokens. This provides the main tables and curves.\n\n\\paragraph{E2: Hyperparameter sanity (one-dimensional).}
Fix all settings as in E1 and sweep \emph{only} the sublinear dampening exponent for length-based methods, $\alpha\in\{0.5,0.75,1\}$, on CountDown. This mirrors the $\Delta L$ hyperparameter analysis and tests the ``use long responses more'' hypothesis.
